# 1.11 헤더파일 만들기

## 헤더파일

- 헤더와 cpp로 쪼개서 구현하고 include 하는 방식으로 진행된다.

### add.h에 구현 포함 (without add.cpp)

> 아래와 같이 구현하면 `add()` 함수 구현이 `main.cpp` 에도, `foo.cpp` 에도 중복으로 들어가면 `링커 오류`가 발생

- main.cpp
- add.h

```cpp
// add.h
#pragma once

int add(int a, int b) {
    return a + b;
}
```

### main.cpp, add.h, add.cpp

> `add.h`는 선언만, `add.cpp`는 구현만 담아 분리하는 방식이 정석이다.
> 여러 `.cpp` 파일에서 `add.h`를 `#include` 해도 구현이 중복되지 않아 링커 오류가 발생하지 않는다.
> 큰 프로젝트에서는 파일이 수십~수백 개가 되므로 이렇게 선언과 구현을 분리하는 것이 필수다.
