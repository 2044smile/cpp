# 1.12 헤더 가드가 필요한 이유

## 링커 에러

> 정의(Definition)는 하나만 존재해야 한다

```txt
소스 코드 (.cpp)
      ↓
컴파일
      ↓
오브젝트 파일 (.o / .obj)
      ↓
링킹 (Linking)
      ↓
실행 파일
```

- 링커 에러는 여러 개의 오브젝트 파일을 연결하는 과정에서 함수나 변수의 정의를 찾지 못하거나, 같은 정의가 여러 번 존재할 대 발생하는 에러

### extern

```cpp
// config.h > 선언
extern int fps;

// config.cpp > 정의
int fps = 30;
```

### inline 변수 (C++17)

```cpp
// 여러 translation unit에 정의가 있어도 컴파일러가 하나로 합쳐준다
inline int fps = 30;
```

## 헤더 가드

- `#include` 는 파일 내용을 그대로 복사해서 붙여넣는 것과 동일
- 컴파일러 입장에서는 A.h 내용이 여기에 그대로 들어옴
- 같은 헤더가 여러 번 `include` 되면 중복 정의 문제가 발생
- 요즘은 대부분 `#pragma once` 를 사용
    - 더 간단
    - 실수 없음
    - 대부분 컴파일러 지원

### 상황 1: 같은 헤더가 여러 번 include됨

#### A.h

```cpp
int add(int a, int b);
```

#### main.cpp

- 일반 함수 선언은 괜찮을 수 있지만 class / struct / 변수 정의는 문제가 된다

```cpp
#include "A.h"
#include "A.h"

// int add(int a, int b);
// int add(int a, int b);
```

## 복습

- 같은 헤더가 여러 번 include되면 재정의 오류가 발생합니다.
- `#pragma once` 또는 전통적인 `#ifndef` 가드로 방지합니다.

```cpp
// 방법 1: pragma once (현대적, 권장)
#pragma once

// 방법 2: ifndef 가드 (전통적, 이식성 최고)
#ifndef ADD_H
#define ADD_H
int add(int a, int b);
#endif

// 전역 변수 헤더에 넣기
extern int fps;   // 헤더: 선언만
int fps = 30;     // cpp: 정의

// C++17: inline 변수
inline int fps = 30;    // 헤더에서 직접 정의 가능
```

### inline

- 설정값/상수/작은 변수

```cpp
inline constexpr int MAX_SIZE = 100;
inline const char* APP_NAME = "Test";
inline int default_port = 8080;
```

### extern

- 상태가 중요하거나 큰 객체

```cpp
int db_connection_count;
UserManager global_manager;
HugeCache system_cache;
```
