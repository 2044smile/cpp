# 1.9 기본적인 서식 맞추기

## Google C++ Style Guide 또는 LLVM Style을 팀 단위로 통일해 사용하는 것을 권장합니다

```cpp
// 나쁜 예 [X]
int add(int x, int y){return x+y;};

// 좋은 예 [O]
int add(int x, int y) {
    return x + y;
}
```

## 심화

- clang-format 도구를 사용하면 코드 서식을 자동으로 통일할 수 있습니다
- CI 파이프라인에 통합하면 PR 단계에서 서식 오류를 자동 차단합니다

## 최소화, 개발하는 사람들과 맞춰라

## 붙여서 작성하지 말라

```cpp
// [X]
int add(int x, int y) {return x+y;}
// [O]
int add(int x, int y) {return x + y;}

// [X]
int sub(int x, int y) { 
return x - y; 
}
// [O]
int sub(int x, int y) { 
return x - y; 
}
```

## 하드 코딩

- 프로그래머가 코드에서 변수값을 리터럴 같이 고정된 값으로 직접 대입해주는 방식

## 소프트 코딩

- 프로그램 실행 중에 사용자의 입력이나 외부 파일, 인터넷 통신 등으로 데이터를 가져오는 방식
