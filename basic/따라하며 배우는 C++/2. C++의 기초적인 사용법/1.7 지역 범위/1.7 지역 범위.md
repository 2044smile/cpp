# 1.7 지역 범위

- 중괄호 안에 선언된 변수는 그 블록 안에서만 사용 가능

## 참고

- 지역 변수 영역을 벗어나면 사용할 수 없게 됩니다.
- 지역 변수가 차지하고 있던 메모리는 그 지역 변수가 영역을 벗어날 때 스택(stack) 메모리로 반납됩니다.
- 반납된 메모리는 다음 지역 변수가 사용할 수 있도록 대기합니다.

## 실무에서는 vector를 많이 사용한다

`vector<int> v(1000000);`

- 자동 메모리 관리
- 안전함
- 예외에 강함

## 심화

- 전역 변수는 데이터 세그먼트(RAM; 메인 메모리)에 저장
- 지역 변수(Stack)는 스택에 저장
    - 스택은 빠르지만 크기가 제한적(보통 1~8MB)이므로 대형 배열은 heap 또는 전역으로 선언
- 동적 할당은 힙(Heap)에 저장

```txt
높은 주소
┌───────────────┐
│ Stack(1~8MB)  │ ← 지역변수
│               │
├───────────────┤
│ Heap          │ ← new, malloc
│               │
├───────────────┤
│ BSS           │ ← 초기화 안 된 전역변수
├───────────────┤
│ Data Segment  │ ← 초기화된 전역변수
├───────────────┤
│ Text Segment  │ ← 코드
└───────────────┘
낮은 주소
```

```cpp
#include <iostream>
using namespace std;

/*
1. 전역 변수 (Global Variable)
→ 프로그램 시작부터 종료까지 존재
→ 데이터 세그먼트(Data Segment)에 저장
*/
int global_var = 100;

/*
큰 배열을 전역으로 선언하면
Stack Overflow를 피할 수 있음
*/
int global_arr[1000000];

void exampleFunction() {
    /*
    2. 지역 변수 (Local Variable)
    → 함수가 호출될 때 생성
    → 함수가 끝나면 사라짐
    → Stack 영역에 저장
    */

    int local_var = 10;

    /*
    너무 큰 배열을 지역 변수로 선언하면
    Stack Overflow 발생 가능
    */

    // int local_arr[1000000]; // 위험할 수 있음

    cout << "local_var: " << local_var << endl;
}

int main() {
    /*
    3. 동적 할당 (Dynamic Allocation)
    → new 로 생성
    → Heap 영역에 저장
    → 직접 delete 해줘야 함
    */

    int* heap_var = new int(50);

    int* heap_arr = new int[1000000];

    cout << "global_var: " << global_var << endl;
    cout << "heap_var: " << *heap_var << endl;

    exampleFunction();

    /*
    Heap 메모리는 반드시 해제
    */

    delete heap_var;
    delete[] heap_arr;

    return 0;
}
```
