# 1.14 전처리기와의 첫 만남

## define 매크로

- 컴파일 전(전처리 단계)에 단순 텍스트 치환
- define이 선언된 파일 안에서만 작용
- Preprocessor (전처리)
- Compiler
- Linker

### 주로 쓰이는 곳

#### 헤더 중복 방지

- C++에서 가장 많이 보이는 매크로, 요즘은 `#pragma oncne` 로 사용

```cpp
#ifndef MY_HEADER_H
#define MY_HEADER_H

// header 내용

#endif
```

#### 플랫폼 분기

```cpp
#ifdef _WIN32
// Windows 코드
#else
// Linux 코드
#endif
```

## 매크로 대신 요즘 쓰는 것

### 상수

- constexpr
    - 컴파일 시점에 값(상수) 결정
    - Scope 존재
    - #define은 #include 하면 사용할 수 있음

```cpp
constexpr int NUMBER = 9;

void func()
{
    constexpr int NUMBER = 9;
    // func 안에서만 사용 가능
}

// namespace도 적용 가능
namespace config
{
    constexpr int BUFFER_SIZE = 1024;
}
```

### 함수 

- inline 의미
    - 함수 호출 대신 코드 삽입
    - 함수라서 디버깅 가능
    - Scope 존재

```cpp
inline int max(int a, int b)
{
    return a > b ? a : b;
}
```

## 복습

- 전처리기는 컴파일 전에 소스코드를 텍스트 치환하는 단계입니다.
- #include, #define, #ifdef 등의 지시어를 처리합니다.

```cpp
#ifdef _WIN32
    cout << "Windows";
#elif __linux__
    cout << "Linux";
#endif

// #define 상수 -> constexpr 권장 (타입 안전)
#define OLD_PI  3.14        // 나쁜 예: 타입 없음
constexpr double PI = 3.14  // 좋은 예: 타입 + 스코프

// 인라인 함수 - 매크로 함수 대체
#define MAX(a, b) ((a)>(b)?(a):(b)) // 나쁜 예
inline int max(int a, int b) { return a > b ? a : b; }
```

## 심화

- 현대 C++에서는 #define 매크로 대신 constexpr, inline, template을 사용하세요.
- 매크로는 디버거가 추적하지 못하고 스코프와 타입 안전성이 없습니다.
