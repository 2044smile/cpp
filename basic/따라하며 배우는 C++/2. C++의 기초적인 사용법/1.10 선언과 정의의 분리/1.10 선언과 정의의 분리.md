# 1.10 선언과 정의의 분리

## 컴파일러

- line 1 부터 순서대로 읽는다
- main() 함수가 위에 선언되어 있고, 다른 함수를 main() 함수 아래 선언하고 실행하면 에러 발생

## forward declaration(전진 선언)

- Forward Declaration을 통해 함수 시그니처를 미리 알리고, 정의는 나중에 작성할 수 있습니다.
- 대규모 프로젝트에서 헤더/소스 파일 분리의 근간입니다.

```cpp
#include <iostream>

using namespace std;

// 전진 선언 (헤더에 위치 .h)
int add(int a, int b); // *
int multiply(int a, int b); // *
int subtract(int a, int b); // *

int main()
{
    cout << add(1, 2) << endl;
    cout << multiply(3, 4) << endl;
    cout << subtract(5, 6) << endl;

    return 0;
}

// definition 정의 (소스 파일 하단 또는 별도 .cpp)
int add(int a, int b){
    return a + b;
}
int multiply(int a, int b)
{
    return a * b;
}
int subtract(int a, int b)
{
    return a - b;
}
```

## 심화

### ODR(One Definition Rule)

- 하나의 정의는 전체 프로그램에서 단 한 번만 존재해야 합니다.
- 함수 정의를 헤더에 넣을 경우 반드시 inline을 붙이세요
    - inline은 여러 파일에 같은 정의가 있어도 괜찮다
