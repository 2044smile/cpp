# 2.5 부동소수점 수 floating point numbers

|영역(Category)|형(Type)|최소 크기|전형적인 크기|
|--|--|--|--|
|부동소수점 (Floating point)|float|4byte|4byte|
||double|8byte|8byte|
||Long double|9byte|8,12 or 16byte|

## 부동소수점(실수)

|부호(sign)|지수(exponent)|가수(mantissa)|
|--|--|--|
|0(양수)|00000111|11000000000000000000000|
|1(음수)|2⁰+2¹+2²=7|2⁻¹+2⁻²=0.5+0.25=0.75|

### 정규화란

- 부호(1) + 지수(8) + 가수(23) = 32bit
- 정규화 덕분에 가수 23비트를 최대한 활용 가능
- 컴퓨터가 숫자를 빠르고, 작게, 정확하게 처리하기 위해서입니다
- 저장 공간 절약: 앞서 말했듯 정수부가 항상 `1`이면 그 `1`은 저장할 필요가 없습니다

## 곱하기, 더하기, 빼기는 문제가 없지만 사용하다보면 한계가 생길 수 있다

## 부동소수점의 툭수값들

- 정수였다면 0으로 나누면 프로그램이 그냥 죽습니다(crash) 하지만 float/double은 IEEE 754 표준에 따라 특수값으로 처리합니다
    - `inf` (Infinity) 양의 무한대
        - 수학적으로 "0으로 나누면 무한대" 라는 개념
    ```cpp
    5.0 / 0.0 = inf  // 양수 % 0
    ```

    - `-inf` 음의 무한대
    ```cpp
    -5.0 / 0.0 = -inf  // 음수 % 0
    ```

    - `nan` (Not a Number)
        - "정의할 수 없는 값"
        - 0%0은 수학적으로도 답이 없습니다
        - 컴파일러/플랫폼마다 `-nan`으로 출력될 수 있고, `nan`으로 출력될 수도 있지만 의미는 동일
    ```cpp
    0.0 / 0.0 = nan
    ```
