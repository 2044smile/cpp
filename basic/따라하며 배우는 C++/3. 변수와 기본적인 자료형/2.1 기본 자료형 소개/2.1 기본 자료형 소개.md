# 1. 기본 자료형 소개

## 문자열이 어떻게 컴퓨터에 저장이 될까?

```cpp
char a = 'H'; // 문자는 1 byte
// 'H' -> ASCII -> 72 -> 이진수 -> 01001000
int i = 1; // 정수는 4 byte
// 00000000 00000000 00000000 00000001
```

- 주소는 계속 변경된다

## Group

### Character types

- `char`: 8bits -> 1byte
- `char16_t`: 16bits -> 2byte
- `char32_t`: 32bits -> 4byte
- `wchar_t`: largest character set

### Integer types (signed: 부호 있음)

- `signed char`: 8bits
- signed `short` int: 16bits
- signed `int`: 16bits
- signed `long` int`: 32bits
- signed `long long` int: 64bits

### Integer types (unsigned: 부호 없음)

> 같은 메모리 크기로 2배의 더 큰 양수를 표현 가능!

- `unsigned char`: same size as their signed counterparts
- `unsigned short` int: same size as their signed counterparts
- `unsigned` int: same size as their signed counterparts
- `unsigned long` int: same size as their signed counterparts
- `unsigned long long` int: same size as their signed counterparts

#### 언제 사용할까?

1. 배열인덱스, 크기, 개수

```cpp
unsigned int size = 100;  // 배열 크기는 음수일 수 없음
unsigned int count = 0;  // 개수도 음수 불가
for (unsigned int i = 0; i < sizeof; i++) { }
```

2. 나이, 점수 등 음수가 불가능한 값
3. 비트 연산 (비트마스크, 플래그)
4. 메모리 주소, 파일 크기

### Floating-point types (부동소수점)

- `float`: 4byte
- `double`: 8byte
- `long double`: double보다 메모리 사용량이 크다

### Boolean type

- `bool`

### Void type

- `void`: no storage

### Null pointer

- `decltype(nullptr)`

## Python vs C++

- Python은 동적 타이핑(Dynamic Typing)언어라서 변수에 어떤 타입이든 올 수 있다. 이게 편리하지만 메모리 측면에서는 비용이 발생한다

### Python이 메모리를 많이 쓰는 이유

- Python의 모든 변수는 사실 객체(Object)에 대한 참조(Reference) 입니다. 단순한 정수 하나도 내부적으로 여러 메타데이터를 포함합니다

```python
import sys

# Python에서 정수 하나의 메모리
x = 1
print(sys.getsizeof(x))  # 28 bytes (타입 정보 + 참조 카운터 + 값)

# C++에서는 int는 4 bytes
# Python은 같은 숫자를 저장하는데 7배 더 사용
```

### C++ 왜 세분화가 필요한가?

1. 메모리 절약: 0~100 사이 값만 저장한다면 `char` (1byte)로 충분한데 , `int` (4byte) 쓰면 3byte 낭비
2. 성능: **메모리를 적게 쓰면 CPU 캐시에 더 많은 데이터가 들어가서 빠름**
3. 임베디드/게임: 메모리가 제한적인 환경에서는 1byte도 아껴야 함
