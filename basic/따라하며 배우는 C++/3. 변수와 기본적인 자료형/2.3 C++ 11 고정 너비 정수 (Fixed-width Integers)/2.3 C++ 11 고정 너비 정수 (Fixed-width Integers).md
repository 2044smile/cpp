# 2.3 C++ 11 고정 너비 정수 (Fixed-width Integers)

## 왜 Fixed-width Integers가 필요한가?

- 크기를 확실히 보장하기 위해 Fixed-width Integers가 등장

```cpp
// 일반 int의 문제점
int x = 5;
// 이 int가 몇 bytes인지 플랫폼마다 다르다! 즉, 코드를 다른 환경에서 실행하면 다르게 동작할 수 있음
/// Windows 32bit -> 2 bytes
/// Windows 64bit -> 4 bytes
```

## Fixed-width Integer 종류 전체 정리

> int_fast8_t   = "나는 빠름이 목표, 메모리는 좀 써도 됨" -> 고성능 연산(게임, AI) 속도가 중요
> int_least8_t  = "나는 절약이 목표, 좀 느려도 됨 -> 임베디드(IoT, 센서) 메모리 아껴야 함

### 1. 정확한 크기 보장 (`int8_t` 계열)

```cpp
std::int8_t     a = 100;  // 정확히 8bits = 1byte (-128 ~ 127)
std::int16_t    b = 100;
std::int32_t    c = 100;
std::int64_t    d = 100;

// unsinged 버전
std::uint8_t    e = 200;  // 0 ~ 255
std::uint16_t   f = 100;
std::uint32_t   g = 100;
std::uint64_t   h = 100;
```

### 2. 가장 빠른 처리 (`int_fast8_t` 계열)

> `최소 N bits이면서, CPU가 가장 빠르게 처리하는 크기`를 자동 선택합니다

```cpp
std::int_fast8_t    fi(5);
// 최소 8bit 이상이면서 이 CPU에서 제일 빠른 타입 써줘

std::int_fast16_t    fi(5);
std::int_fast32_t    fi(5);
std::int_fast64_t    fi(5);
```

### 3. 가장 작은 크기  (`int_least8_t` 계열)

> `최소 N bits를 만족하는, 가장 메모리를 적게 쓰는 타입`을 선택

```cpp
std::int_least8_t   fl(5);  // 최소 8비트 -> 보통 8비트
std::int_least64_t  fl2(5);
// 메모리 최우선
```
