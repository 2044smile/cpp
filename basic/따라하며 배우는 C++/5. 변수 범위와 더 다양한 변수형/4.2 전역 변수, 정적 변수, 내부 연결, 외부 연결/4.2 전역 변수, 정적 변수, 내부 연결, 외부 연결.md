# 4.2 전역 변수(Global Variable), 정적 변수(Static Variable), 내부 연결(Internal Linkage), 외부 연결(External Linkage)

> 전역 변수는 가급적 사용을 안해야 한다

## 1. 전역 변수

> 함수 밖에 선언된 변수, 프로그램 어디서든 접근 가능하다

```cpp
#include <iostream>
using namespace std;

int g_apple = 5;    // 전역 변수 (관례상 g_ 접두사 사용)

void doSomething() {
    // int a = 1;   // 매번 1로 초기화
    g_apple += 1;
    cout << g_apple << endl;    // 6
}

int main()
{
    cout << g_apple << endl;    // 5
    doSomething();
    cout << g_apple << endl;    // 6
}
```

### 지역 변수 vs 전역 변수

```cpp
int a = 1;      // 전역 변수

int main()
{
    int a = 2;
    cout << a << endl;      // 2 <- 지역 변수 우선
    cout << ::a << endl;    // 1 <- :: 붙이면 전역 변수 접근
}
```

## 2. 정적 변수

> `static` 키워드를 붙이면 함수가 끝나고 값이 유지된다

```cpp
void doSomething() {
    static int a = 1;   // 처음 한 번만 초기화
    a += 1;
    cout << a << endl;
}

int main()
{
    doSomething();  // 2
    doSomething();  // 3
    doSomething();  // 4
}
```

## 3. 내부 연결

> `static`을 전역 변수에 붙이면 같은 파일 안에서만 접근 가능

> Internal Linkage = 자기 파일 안에서만 사용, 다른 파일에서 접근 X

```cpp
// a.cpp
static int g_a = 1; // 이 파일 안에서만 사용 가능

// b.cpp
extern int g_a;     // 접근 불가
```

## 4. 외부 연결 (External Linkage)

> `extern` 키워드로 다른 파일의 변수/함수를 가져와서 사용 가능

```cpp
// a.cpp
int g_a = 1;
```
